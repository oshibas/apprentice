# Cookie とセッションについて説明できる

## 1. ステートレス

>HTTP は特徴としてステートレスであると言われます。ステートレスとは何か、プログラミング初心者にわかるように説明してください。

「サーバーがクライアントの状態を保持しないこと」を意味します。HTTP 通信では、各リクエストが独立しており、前のリクエストと次のリクエストの間に関連性がないということです。

## 2. Cookie とセッション

>HTTP がステートレスであることを考えると、ログイン状態を保持することができません。しかし現実のアプリケーションではログイン状態を保持できないと大変不便です。そこでログイン状態を保持するために Cookie とセッションが使われます。以下について説明してください。

- Cookie とセッションとはそれぞれ何か？

1. Cookie: クライアント側（Web ブラウザ）に保存される小さなデータファイルです。Web サーバーは、レスポンスヘッダの Set-Cookie フィールドを使用して、クライアントに Cookie を送信します。クライアントは、その後のリクエストで、リクエストヘッダの Cookie フィールドに Cookie を含めて送信します。
2. セッション: サーバー側に保存されるデータです。サーバーは、セッション ID を発行し、それをクライアントに送信します。クライアントは、その後のリクエストで、セッション ID を含めて送信します。サーバーは、セッション ID を使用して、対応するセッションデータを取得します。

- Cookie とセッションの違いは？

保存場所が異なることです。Cookie はクライアント側に保存されますが、セッションはサーバー側に保存されます。

- Cookie とセッションを利用してどのようにログイン状態を実現しているのか？


1. ユーザーがログイン情報（ユーザー名やパスワード）を入力して送信する。
2. サーバーは、ログイン情報が正しいかどうかを確認する。
3. ログイン情報が正しい場合、サーバーはセッション ID を発行し、それを Cookie に保存してクライアントに送信する。
4. クライアントは、その後のリクエストで、Cookie に含まれるセッション ID を送信する。
5. サーバーは、セッション ID を使用して、対応するセッションデータ（ログイン状態など）を取得する。


## 3. Cookie の確認


>ブラウザで Cookie を確認してみましょう。`www.google.com` にログインした状態でアクセスし、Cookie を確認してください。

Google Chrome の場合: [設定] > [プライバシーとセキュリティ] > [Cookieと他のサイトデータ] > [すべてのサイトデータと権限を表示] の順に選択します。
